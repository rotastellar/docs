# Changelog

All notable changes to the RotaStellar API.

## 2026-03-10 ‚Äî Orbital Resilience Portfolio

### New: 5 Orbital Resilience Inventions

- **I-1: Orbital Compute Primitives** ‚Äî 3 new step types (eclipse, window, pass) for orbit-aware workload scheduling with energy budgets, adaptive quality tiers, and ground contact decomposition
- **I-2: Predictive Pareto Planning** ‚Äî Multi-objective optimization across 4 objectives (latency, energy, reliability, quality) with dominance filtering and 4 relaxation types
- **I-3: Constellation DAG** ‚Äî Multi-satellite workflow orchestration with time-expanded Dijkstra ISL routing, 5-phase placement algorithm, and k-d tree spatial indexing
- **I-4: Eclipse-Boundary Preservation** ‚Äî Hazard-predictive checkpointing using Keplerian+J2 propagation, SAA detection, and thermal prediction with automatic checkpoint scheduling
- **I-5: Orbital Compute Unit** ‚Äî Standardized capacity measurement across 6 satellite bus classes (1U CubeSat through SmallSat) with confidence and cost models

### New: Simulation Sessions (Sim v1.1.0)

- **Stateful sessions** ‚Äî Create multi-satellite simulation sessions with KV-backed state persistence
- **Subsystem tracking** ‚Äî Battery charge/discharge, solar power, temperature, memory, CPU per satellite
- **ISL link quality** ‚Äî Distance-based quality model with eclipse penalty and effective bandwidth
- **Fault injection** ‚Äî 5 fault types (power_loss, thermal_exceedance, radiation_upset, comms_failure, isl_degradation) with natural detection (SAA, thermal, critical battery)

### New: Agent Constellation Mode

- **ConstellationState** ‚Äî DAG step tracking with assignment, execution, completion, and failover state machine
- **ISL transfer lifecycle** ‚Äî Hop-by-hop quality metrics, bandwidth, and latency reporting
- **Automatic failover** ‚Äî Battery <10%, temperature >75¬∞C, or compute unavailable triggers step reassignment
- **Event enrichment** ‚Äî All constellation events include actual satellite state (battery, temperature, position, eclipse)

### New: Monitoring & Operations

- **CAE metrics endpoint** ‚Äî Per-endpoint latency percentiles (p50/p95/p99), request counts, status distribution
- **Consolidated API directory** ‚Äî 39 public endpoints documented across Intelligence, CAE, Sim, and Agent Protocol services
- **Pareto + Constellation UI** ‚Äî Interactive scatter chart for fleet-level trade-off analysis
- **Monitoring dashboard** ‚Äî Admin dashboard with real-time CAE metrics visualization

## 2026-03-05 ‚Äî CAE v1.0

### New: Constraint-Aware Execution

- **CAE API** ‚Äî POST a workload + satellite ID, get a physically-accurate execution plan computed from real orbital mechanics
- **5 built-in presets:** on-board ML inference, split learning, earth observation with QA, federated learning, resilient store-and-forward
- **Custom workload DAGs** ‚Äî define arbitrary step pipelines with dependencies, resource requirements, and security policies
- **4-phase planner:** topological sort, compute placement, transfer insertion, window scheduling
- **Real orbital mechanics:** SGP4 propagation, eclipse detection, 12-station ground network (KSAT, NASA, AWS)
- **Simulated execution events:** full event stream with transfer, checkpoint, and security events

## Versioning

The API uses date-based versioning in the URL path:

```
https://api.rotastellar.com/v1/...
```

Breaking changes will be released as new versions with migration guides.

## Deprecation Policy

- Deprecated features are announced 6 months before removal
- Deprecated endpoints return `X-Deprecated: true` header
- Migration guides are provided for all breaking changes
